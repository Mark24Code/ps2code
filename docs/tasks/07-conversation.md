# Task 07 — 对话界面(左对话列表 + 中对话 + 右预览)

## 目标
项目内的主工作区,类 codex 三栏布局。

## 布局
- **最左:对话列表**(围绕当前项目/设计稿名)。可新建、删除对话。
- **中间:对话区**。展示对话历史、用户输入、Agent 日志、交互确认气泡。
  - 顶部**高亮 @设计稿文件**:锁定不可改;要换设计稿需新建对话。
  - 底部输入框 + 设置勾选框(见 Task 08):裁剪 / 1x / 2x(默认2x)/ 导出路径。这些每次作为上下文传给 Agent。
- **右侧:预览区**。展示本对话 tmp 目录下的导出处理结果(图片网格)。用户确认后可导出(Task 08)。

## 数据流
- 进入对话:读 conversations + messages(Task 02)、psd 图层树(Task 03)。
- 发送:调 Agent(Task 05),流式渲染 `agent:stream` 事件为消息/日志/确认。
- 每对话独立 tmp 目录(`app.getPath('temp')/ps2code/<conversationId>`),预览读取其中产物。

## 视觉
- 与主界面一致的黑白灰 + 商务蓝,无渐变。对话气泡区分 user/assistant/tool/确认。

## 验收
- 新建/切换/删除对话正常;@设计稿锁定生效。
- 一次对话产出的预览图只显示在该对话右侧。

## 依赖
- Task 05(agent)、Task 06(进入项目)
