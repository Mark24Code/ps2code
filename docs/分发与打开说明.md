# PS2Code 分发与在其它 Mac 上打开

PS2Code 使用**本地 ad-hoc 签名**(未经 Apple 公证)。这样无需付费开发者账号即可打包,
产物能拷到其它 Mac 使用;但首次打开时 macOS 的 Gatekeeper 会拦一次,按下面任一方式放行即可。

## 打包

```bash
npm run dist:mac        # 产出 dist/PS2Code-<版本>-arm64.dmg
```

- 已配置 `mac.identity: '-'`(ad-hoc 签名),不弹钥匙串、不联网公证,不会卡住。
- 若仍想完全不签名:`CSC_IDENTITY_AUTO_DISCOVERY=false npm run dist:mac` 并把 identity 改回 null。

## 在其它 Mac 上打开(三选一)

### 方式一:系统设置放行(最推荐,无需命令行)
1. 双击 PS2Code,出现「无法打开,因为 Apple 无法检查其是否包含恶意软件」——先点「完成/取消」。
2. 打开 **系统设置 → 隐私与安全性**,向下滚动到「安全性」区域。
3. 会看到「已阻止使用 “PS2Code”…」,点右侧 **「仍要打开」**。
4. 再次弹窗时点 **「打开」**。之后每次都能正常双击打开。

### 方式二:右键打开
在 Finder 里**右键(或按住 Control 点击)PS2Code → 打开**,在弹窗里再点「打开」。

### 方式三:命令行去隔离(兜底)
```bash
./scripts/unquarantine.sh /Applications/PS2Code.app
# 等价于:
xattr -dr com.apple.quarantine /Applications/PS2Code.app
```

> 说明:以上是无 Apple 开发者账号时的固有限制。若将来要做到「双击零提示、任意 Mac 直接打开」,
> 需 Developer ID 证书签名 + notarytool 公证(见 docs/tasks/10-e2e-build.md)。

## Windows
`npm run dist:win` 产出 NSIS 安装包;未签名时首次运行 SmartScreen 可能提示,点「更多信息 → 仍要运行」。
